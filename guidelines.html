File structure

{% highlight scss %} // Vendors // bower:scss // endbower // Utils @import "utils/variables"; @import "utils/utils"; // Base @import "base/base"; @import "base/typo"; // Components @import "components/buttons"; @import "components/icons"; @import "components/navigation"; // Layout @import "layout/header"; @import "layout/main"; @import "layout/footer"; // Pages @import "pages/home"; // Anything that can be categorized later @import "todo"; {% endhighlight %}

vendors/

Use Wiredep to inject our dependencies directly into style.scss.

utils/

Here are the files that have functions, mixins, extends, or classes that we use more than once in different folders.

base/

Basic styles like html, body, p, and a tags.

components/

UI components that can be used to build layouts.

layout/

Header, footer, sidebar and other elements that makes the site structure.

Layouts are built with components. We code the header with the .site-logo, .site-navigation, and .social-icons components and make them work together inside the _header.scss layout.

pages/

Styles for specific pages:

  • _home.scss
  • _contact.scss
  • _portfolio.scss

Formatting

  • For indentation use 2 spaces, not 4 spaces or tabs.
  • 1 selector or parameter per line.
  • No whitespace at the end of the line.
  • Put a blank line between rule declarations.
  • Add single a space after :.
  • Add a space before { in rule declaration.
  • Closing } on its own line.
  • When listing multiple selectors associated with the same CSS rule, add a new line after each ,.
  • Use hyphens for class names, not underscores or camelCase.
  • No spaces within () for urls.
  • Use hex color codes (#5c8fb3) unless using rgba()
  • Use em over px, unless unavoidable.
  • Don’t specify units for 0 values.
  • Omit leading zeros in decimal values.
  • Use double quotes instead of single quotes.
  • Always have a semicolon at the end of each declaration.
  • Avoid the use of !important.
{% highlight css %} /* Bad */ .site_header { background-color : #FFF; background-image : url( ‘/img/background.jpg’ ); display : block; position : relative; width : 100%; } .site-logo, .site-menu { margin-right: 0.618em; padding-top: 16px; padding-right: 0px; } /* Good */ .site-header { position: relative; display: block; width: 100%; background-color: #fff; background-image: url(“/img/background.jpg”); } .site-logo, .site-menu { padding: 1em 0; margin-right: .618em; } {% endhighlight %}

Comments

Comments start with a capital first letter, but don’t require a period at the end, unless you’re writing full sentences. There must be a single space between the comment token and the comment text.

On CSS, when comments take up multiple lines, use a * at the beginning of each line. There must be a space between the first word and the * on each line.

{% highlight css %} /* Good single line comment */ /* * Good example of a multi-line comment. * If your comment takes up multiple lines, please do this. */ {% endhighlight %}

On SCSS, single and multiple lines use // at the beginning of each line.

{% highlight scss %} // Good single line comment // // Good example of a multi-line comment. // If your comment takes up multiple lines, please do this. // {% endhighlight %}

SCSS general formatting

Avoid unnecessary nesting.

{% highlight scss %} // Bad .site-header { .logo { img { float: left; margin-right: 1em; } } } // Good .site-logo-img { float: left; margin-right: 1em; } {% endhighlight %}

Order extend/inheritance by:

  • @extends
  • @includes
  • Regular styles
  • Nested pseudo classes and pseudo elements
  • Nested selectors
{% highlight scss %} // Bad .site-header { background-color: #eee; .site-logo { float: left; display: block; } &:hover { background-color: #fff; } &::before { background-color: #ccc; } @extend %clearfix; @include .module } // Good .site-header { @extend %clearfix; @include .module background-color: #eee; &:hover { background-color: #fff; } &::before { background-color: #ccc; } .site-logo { float: left; display: block; } } {% endhighlight %}

Don’t hardcode colors, use variables instead.

{% highlight scss %} // Bad .entry-title { color: #1d4178; } // Good // _variables.scss $brand-primary-color: #1d4178; // _main.scss .entry-title { color: $brand-primary-color; } {% endhighlight %}

Nest your media queries.

{% highlight scss %} // Bad .site-header { float: left; width: 50%; } @media (min-width: 48em) { .site-header { float: none; width: 100%; } } // Good .site-header { float: left; width: 50%; @media (min-width: 48em) { float: none; width: 100%; } } {% endhighlight %}
